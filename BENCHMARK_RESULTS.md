## 2. 🔥 explode+implode 优化：Before vs After

| 编译器 | 优化前 (ns) | 优化后 (ns) | 提速 | vs PHP |
|--------|------------|------------|------|--------|
| **TCC** | 18,675,800 | **9,793,500** | **1.9x** ⚡ | 0.16x → 0.30x |
| **GCC -O2** | 15,797,900 | **5,355,600** | **2.9x** ⚡ | 0.18x → 0.54x |
| **Clang -O2** | 12,460,300 | **5,777,700** | **2.2x** ⚡ | 0.23x → 0.51x |

**优化手段**：
- `implode`: O(N²) 逐对拼接 → O(N) 两遍扫描一次分配（类似 ROPE 思路）
- `explode`: 固定 capacity=8 + realloc → 先数分隔符 → 精确容量一次分配

## 6. 总评

### 编译器对比

| 编译器 | 编译速度 | 运行性能 | 推荐场景 |
|--------|---------|---------|---------|
| **TCC** | ⚡ 极快 | 基准 | 开发调试 |
| **GCC -O2** | 中等 | TCC 的 3-10x | 生产 (Linux) |
| **Clang -O2** | 中等 | TCC 的 3-12x | 生产 (跨平台) |

### 关键性能数据 (GCC/Clang -O2 vs PHP)

| 场景 | 倍数 | 
|------|------|
| count+for | **35-36x** 🔥 |
| foreach 1K | **29-32x** 🔥 |
| 嵌套数组读 | **23-27x** 🔥 |
| int key 读取 | **13-18x** |
| array_pop | **8-10x** |
| in_array | **2.0-2.6x** |
| explode+implode | 0.5x |

### 优化成果

| 优化项 | 技术 | 效果 |
|--------|------|------|
| **ROPE 多片段拼接** | PHP ROPE opcode | concat-4: 14x慢 → 6.1x快 |
| **implode 两遍扫描** | 自研 (O(N²)→O(N)) | explode+implode: **2-3x 提速** |
| **explode 精确容量** | 预数分隔符 | 消除所有 realloc |
| **JSON 位图+批量写入** | PHP json_encoder.c | json_encode 接近持平 |
| **数组池预热** | PHP zend_alloc | arr-create 12x慢 → 4.4x快 |

### 剩余优化空间

- **SSO 小字符串**: 24B 内联缓冲区，字符串密集场景 2-3x
- **Arena Allocator**: 函数级 bump 分配器，全局 1.5-3x
- **`$a[]=` 语法支持**: 消除 array_push 函数调用开销
