## 5. 总评

### 编译器对比总结

| 编译器 | 编译速度 | 运行性能 | 推荐场景 |
|--------|---------|---------|---------|
| **TCC** | ⚡ 极快 (<0.1s) | 基准 | 开发调试、快速迭代 |
| **GCC -O2** | 中等 (~0.5s) | TCC 的 **3-10x** | 生产部署 (Linux) |
| **Clang -O2** | 中等 (~0.5s) | TCC 的 **3-12x** | 生产部署 (跨平台) |

### 关键发现

| 场景 | TCC vs PHP | GCC -O2 vs PHP | Clang -O2 vs PHP |
|------|-----------|---------------|-----------------|
| foreach 1K | 3.0x ✅ | **32.5x** 🔥 | **29.9x** 🔥 |
| count+for | 5.7x ✅ | **25.8x** 🔥 | **35.6x** 🔥 |
| int key 读取 | 7.3x ✅ | **18.2x** 🔥 | **14.3x** 🔥 |
| 嵌套数组读 | 6.6x ✅ | **16.8x** 🔥 | **22.4x** 🔥 |
| array_pop | 2.2x ✅ | **8.3x** 🔥 | **10.3x** 🔥 |
| in_array | 0.5x ❌ | **2.1x** ✅ | **2.1x** ✅ |
| array_push | 0.4x ❌ | 0.6x ❌ | 0.6x ❌ |
| explode+implode | 0.2x ❌ | 0.2x ❌ | 0.2x ❌ |

> 💡 **核心洞察**: TCC 下只有 5/10 项优于 PHP，换 GCC/Clang -O2 后 **7/10 项全面反超**。编译器优化对 AOT 编译的性能提升是质变级别的。

### 性能特征

| 优势场景 (GCC/Clang) | 待优化场景 |
|---------------------|-----------|
| foreach/for 大循环 (30-36x) | 数组小对象分配创建 |
| int key 直接访问 (14-18x) | explode/implode 字符串操作 |
| count+for 遍历 (26-36x) | array_merge 合并 |
| 嵌套数组读取 (17-22x) | mixed 类型数组 |
| array_pop (8-10x) | — |

### 优化成果（历史）

| 优化项 | 技术来源 | 效果 |
|--------|---------|------|
| **ROPE 多片段拼接** | PHP ROPE opcode | concat-4 从 14x 慢 → 6.1x 快 |
| **JSON 位图+批量写入** | PHP json_encoder.c | json_encode 从 11x 慢 → 接近持平 |
| **数组池预热** | PHP zend_alloc | arr-create 从 12x 慢 → 4.4x 快 |

### 剩余优化空间

- **SSO 小字符串**: 24 字节内联缓冲区，字符串密集场景 2-3x
- **Arena Allocator**: 函数级 bump 分配器，全局 1.5-3x
- **GCC/Clang 生产化**: 换编译器即可，无代码改动，3-12x
